# Title: v0.0.1-1-1-5 - Conditional Tasker Fields Display

## Summary: 
Implement conditional rendering logic in the Profile component to display tasker-specific fields (skills and hourly rate) only for users with user_type "tasker", while ensuring customer users see only their basic profile information without tasker fields.

### Acceptance Criteria:
- [ ] Conditional rendering checks `profile.user_type === 'tasker'`
- [ ] Tasker section has clear visual separation (heading, styling)
- [ ] Skills displayed as formatted text with fallback to "Not specified"
- [ ] Hourly rate formatted as USD currency (e.g., "$45.00")
- [ ] Null/empty skills displays "Not specified"
- [ ] Null hourly rate displays "Not specified"
- [ ] Customer profile does not show tasker section or empty space
- [ ] Styling maintains consistent spacing for both user types
- [ ] Page re-renders correctly if user type changes (edge case)
- [ ] Manual testing completed with both tasker and customer accounts
- [ ] Visual design matches mockups/specifications

### Test Strategy:

**Types of Tests:**
- Manual testing with different user accounts
- Visual regression testing
- Edge case testing

**Test Cases:**
1. **Tasker with All Fields**: Test tasker with skills and hourly_rate populated
2. **Tasker with Empty Skills**: Test tasker with null/empty skills field
3. **Tasker with Empty Rate**: Test tasker with null hourly_rate field
4. **Tasker with Both Empty**: Test tasker with both fields null/empty
5. **Customer Account**: Verify no tasker section appears for customer
6. **Layout Consistency**: Compare spacing between customer and tasker profiles
7. **Currency Formatting**: Verify various hourly_rate values format correctly ($0.00, $999.99, $1234.56)
8. **Long Skills Text**: Test with very long skills string for text wrapping

**Tools:**
- Multiple test accounts (tasker and customer)
- Manual browser testing
- Chrome DevTools for layout inspection

**Description:**

**File:** `frontend/src/pages/Profile.jsx` (validation of existing code)

The conditional rendering logic to verify:
```jsx
{profile.user_type === 'tasker' && (
  <div className="tasker-info-section">
    <h2>Tasker Information</h2>
    
    <div className="profile-field">
      <label className="profile-label">Skills:</label>
      <span className="profile-value">
        {profile.skills || 'Not specified'}
      </span>
    </div>

    <div className="profile-field">
      <label className="profile-label">Hourly Rate:</label>
      <span className="profile-value">
        {profile.hourly_rate 
          ? `$${parseFloat(profile.hourly_rate).toFixed(2)}`
          : 'Not specified'
        }
      </span>
    </div>
  </div>
)}
```

**Potential refinements based on testing:**

If currency formatting needs improvement:
```jsx
const formatCurrency = (amount) => {
  if (!amount) return 'Not specified';
  return new Intl.NumberFormat('en-US', {
    style: 'currency',
    currency: 'USD',
    minimumFractionDigits: 2,
    maximumFractionDigits: 2
  }).format(amount);
};

// Usage in render:
<span className="profile-value">
  {formatCurrency(profile.hourly_rate)}
</span>
```

If skills need enhanced display (e.g., as tags):
```jsx
const renderSkills = (skills) => {
  if (!skills || skills.trim() === '') return 'Not specified';
  
  // If skills are comma-separated, could render as tags
  const skillList = skills.split(',').map(s => s.trim()).filter(Boolean);
  
  if (skillList.length === 0) return 'Not specified';
  
  return skillList.join(', ');
};

// Usage in render:
<span className="profile-value">
  {renderSkills(profile.skills)}
</span>
```
